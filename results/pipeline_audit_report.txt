2. Issues Found & Fixes Applied
-------------------------------
* Dataloader Label Crashes: 
  Found: Attempting to cast string labels like "hatespeech" or "positive" to float caused ValueError crashes.
  Fix: Updated mm_dataset.py to reliably map string identifiers to their respective numerical label values (-1.0 to 2.0 based on task format) directly inside __getitem__. Unknown or invalid labels are masked dynamically as -1.0.

* Dataloader Read Robustness: 
  Found: Dataloader failed fatally when iterating through files with NaN records or broken paths.
  Fix: Implemented a global try-catch inside __getitem__. Missing image strings or missing files correctly generate a blank image tensor (with `has_image` = False) alongside 0-padded tokens. Crashed/Invalid rows return empty dummy tensors to sustain epochs without interrupting 7-hour runs.

* Split Integrity: 
  Found: Label distribution string matching for split creation was breaking on uncommon subsets.
  Fix: Wrote logic in prepare_dataset.py to accurately stratify datasets to datasets/train.csv (80%), val.csv (10%), and test.csv (10%) seamlessly dropping bad data.

* Execution Restrictions:
  Found: The previous run reverted to CPU compute invisibly on fallback mode.
  Fix: Hard-coded a RuntimeError check to force torch.device("cuda:0"). Added specific VRAM allocation printouts.
